Git基础

git的基本使用及版本控制

本地仓库

基础

1，mkdir创建，cd跳转，pwd显示目录

2，init托管仓库

3，仓库内添加文件

4，add将文件添加到仓库

5，commit提交

6，status查看仓库状态

7，diff查看修改内容

8，更新文件版本

8，log或log --pretty=oneline 查看历史版本

9，reset 回退版本

HEAD 表示当前版本,上一个版本就是 HEAD^ ，上上一个版本就是 HEAD^^ ，当然往上100个版本写100个^比较容易数不过来，所以写成 HEAD~100 。

10，reflog 查看每一次命令

11，rm删除文件

12，checkout用版本库替换工作区版本

远程仓库

基础

SSH Key

open ~/.ssh 打开文件夹

1，网页新建仓库Create a new repo

2，添加远程主机（远程库的名字就是origin）

3，push 推送本地master分支到远程master分支，（-u参数是把两个分支关联起来，以后可以简化命令）

4，之后的提交

克隆远程仓库

1，clone 克隆一个本地库

分支

1，git checkout 创建dev分支，切换到分支(git checkout命令加上-b参数表示创建并切换)

2，git branch 查看当前分支

3，在dev分支上正常提交

4，切换回master分支

5，git merge 把指定分支合并到当前分支

6，删除分支

7，如果分支有冲突，合并时Git会标记出来，需要手动修改

8，普通模式合并分支，–no-ff参数，表示禁用Fast forward
因为本次合并要创建一个新的commit，所以加上-m参数，把commit描述写进去。
